# Kigali City Services Directory

A Flutter mobile application that helps Kigali residents locate and navigate
to essential public services and lifestyle locations including hospitals,
banks, cafés, pharmacies, schools, and public services.

Built with Flutter and Firebase as part of Individual Assignment 2.

## Features

### Authentication

- Email and password signup via Firebase Authentication
- Email verification enforced before app access is granted
- Secure login and logout
- Password reset via email
- User profile automatically created in Firestore on signup

### Directory

- Browse all listings in a shared directory
- Search listings by name in real time
- Filter listings by category
- Results update automatically via Firestore real-time streams

### My Listings

- View only your own created listings
- Create new listings with image upload to Firebase Storage
- Edit your existing listings
- Delete your listings with confirmation dialog
- Changes reflected immediately without manual refresh

### Listing Detail Page

- Full listing information display
- Embedded interactive map via OpenStreetMap (flutter_map)
- Map marker showing exact location from Firestore coordinates
- One-tap Get Directions launching Google Maps navigation

### Map View

- Full-screen interactive map showing all listings as markers
- Horizontal preview cards for quick listing browsing
- Tap any marker to center the map on that location

### Settings

- View and edit your profile information
- Toggle location notifications preference
- Toggle dark mode preference
- All preferences persisted to Firestore in real time

## Firestore Database Structure

### `users` collection

Document ID: Firebase Auth UID

| Field                | Type      | Description                      |
| -------------------- | --------- | -------------------------------- |
| uid                  | String    | Firebase Auth user ID            |
| fullName             | String    | User display name                |
| email                | String    | User email address               |
| phone                | String    | Optional phone number            |
| notificationsEnabled | Boolean   | Location notification preference |
| darkMode             | Boolean   | UI theme preference              |
| createdAt            | Timestamp | Account creation time            |
| lastUpdated          | Timestamp | Last settings update             |

### `places` collection

Document ID: Auto-generated by Firestore

| Field       | Type      | Description                                                            |
| ----------- | --------- | ---------------------------------------------------------------------- |
| title       | String    | Name of the place or service                                           |
| category    | String    | Public Services, Cafés & Dining, Hospitals, Banks, Pharmacies, Schools |
| location    | String    | Physical address                                                       |
| contact     | String    | Phone number                                                           |
| description | String    | Description of services                                                |
| latitude    | Number    | Geographic latitude coordinate                                         |
| longitude   | Number    | Geographic longitude coordinate                                        |
| image       | String    | Firebase Storage download URL                                          |
| createdBy   | String    | Firebase Auth UID of creator                                           |
| timestamp   | Timestamp | Server-generated creation time                                         |
| status      | String    | active or pending                                                      |
| updatedAt   | Timestamp | Last edit timestamp (added on update)                                  |

### Composite Index

A composite index is required on the `places` collection:

- `createdBy` — Ascending
- `timestamp` — Descending

This supports the My Listings query which filters by user and orders
by creation time simultaneously.

## State Management

This application uses the **Provider** package for state management.

### Architecture Overview

```
UI (Consumer / context.read)
        ↓
   ListingProvider / AuthProvider
        ↓
   Service Layer (ListingService, DirectoryService, AuthService...)
        ↓
   Firebase (Firestore / Auth / Storage)
        ↑
   notifyListeners() → rebuilds Consumer widgets
```

### Providers

**ListingProvider**

- Manages `allListings` for the Directory screen
- Manages `myListings` for the My Listings screen
- Exposes `addListing()`, `updateListing()`, `deleteListing()`
- Handles `isLoading` and `error` states
- Subscribes to Firestore streams and calls `notifyListeners()`
  on every update

**AuthProvider**

- Wraps AuthService for login, signup, logout
- Manages loading state during auth operations
- Auth routing handled separately by a `StreamBuilder` on
  `FirebaseAuth.instance.authStateChanges()` in `main.dart`

### Key Rule

No direct Firebase or Firestore calls exist inside UI widgets.
All database interactions go through the service layer and are
exposed to the UI via the Provider state layer.

## Firebase Setup

1. Create a Firebase project at console.firebase.google.com
2. Enable Authentication with Email/Password provider
3. Enable Cloud Firestore in production mode
4. Enable Firebase Storage
5. Download `google-services.json` and place in `android/app/`
6. Run `flutterfire configure` to generate `firebase_options.dart`

### Firestore Security Rules

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

### Storage Security Rules

```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /{allPaths=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```
